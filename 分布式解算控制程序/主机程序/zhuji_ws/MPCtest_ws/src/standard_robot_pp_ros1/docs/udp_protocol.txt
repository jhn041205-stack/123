standard_robot_pp_ros1 UDP protocol description

1. IMU phi sender

Purpose:
The node imu_udp_sender_node subscribes to ROS topic serial/imu, converts the IMU quaternion to yaw,
and sends phi to a configured UDP target at 50 Hz.

Default ROS parameters:
- imu_topic: serial/imu
- target_ip: 127.0.0.1
- target_port: 5000
- send_hz: 50.0

UDP payload format:
PHI,<timestamp_ms>,<phi_rad>

Example:
PHI,1773829000123,1.570796

Field description:
- timestamp_ms: ROS timestamp in milliseconds
- phi_rad: yaw angle in radians


2. UDP command receiver

Purpose:
The node udp_command_receiver_node listens on a configured UDP port, receives speed, leg length, and
angular velocity, then publishes ROS commands for the lower controller bridge.

Default ROS parameters:
- listen_ip: 0.0.0.0
- listen_port: 6000
- cmd_vel_topic: cmd_vel
- cmd_leg_length_topic: cmd_leg_length
- cmd_gimbal_topic: cmd_gimbal_joint

Accepted UDP payload format:
CMD,<vx>,<leg_length>,<wz>

Also accepted:
<vx>,<leg_length>,<wz>

Example:
CMD,1.2,0.18,0.5

Field description:
- vx: forward speed, unit m/s
- leg_length: leg length command, unit m
- wz: yaw angular velocity, unit rad/s

ROS outputs generated by udp_command_receiver_node:
- cmd_vel:
  linear.x = vx
  linear.y = 0
  angular.z = wz

- cmd_leg_length:
  data = leg_length

- cmd_gimbal_joint:
  gimbal_pitch_joint = 0
  gimbal_yaw_joint = 0


3. Host side recommendations

- Use UTF-8 plain text UDP payloads exactly as described above.
- One UDP datagram should contain exactly one command frame.
- Recommended sender rate for command packets: 20 Hz to 100 Hz.
- The IMU sender already transmits at 50 Hz by default.

4. Receiver mode

- udp_command_receiver_node uses blocking recvfrom().
- This means the node waits for a UDP packet and does not poll on a timer.
- As soon as one valid UDP packet is received, it is parsed immediately and the ROS command topics
  are published immediately.
